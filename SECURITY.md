# Security & Content Policy

## Content Policy

Yume is an uncensored AI art platform for adults (18+). While we don't censor creative expression, the following content is **strictly prohibited**:

- **Minors / underage characters** (loli, shota, or any depiction of minors)
- **Real person likeness** (deepfakes, celebrity likenesses)
- **Extreme violence / gore**
- **Non-consensual content**

Violations will result in immediate ban and content removal.

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public issue
2. Email: security@yume.wtf
3. Include a detailed description of the vulnerability
4. Allow reasonable time for a fix before public disclosure

## API Security

- All API keys are scoped and rate-limited
- HTTPS required for all endpoints
- API keys should never be exposed in client-side code
- Use environment variables for key storage

## Bug Bounty

We currently do not have a formal bug bounty program, but we appreciate responsible disclosure and will credit researchers who report valid vulnerabilities.
